# Cohort 1 Brief — Technical Self-Hosters

**Size**: 25 testers
**Duration**: 2 weeks from invite

## Selection Criteria

- Self-hosts other open-source tools (n8n, Gitea, Coolify, similar)
- Comfortable with Docker Compose
- Has a spare VPS or home server
- Has expressed interest in agent infrastructure or AI-assisted development

## Testing Brief

1. **Self-host from scratch**: Follow `docs/self-host.md` to get a running instance. Time yourself. Report any step where you got stuck.

2. **Run at least 5 tasks** across at least 2 categories:
   - Coding (bug fix, feature, test generation)
   - Research (competitive analysis, documentation summary)
   - Ops (deployment, infrastructure check)
   - Writing (draft, report, email copy)

3. **For each task**, report:
   - What you asked for
   - What the agent produced (screenshot or copy of Works panel)
   - Whether the output was correct, partially correct, or wrong
   - Whether you understood what the agent did from the UI alone

4. **Report any task** that:
   - Produced an incorrect or missing Work
   - Failed without a clear error message
   - Took longer than you expected with no progress indication
   - Got stuck in a state you couldn't recover from

5. **Report any UI moment** where you:
   - Didn't know what to do next
   - Couldn't tell whether the agent was working or stuck
   - Expected a feature that didn't exist

## Feedback Deliverable

Complete the feedback form within the 2-week window. Attach specific task examples. Testers who don't deliver the form are replaced for Cohort 2.

## What We're Looking For

- Self-host friction: where does the guide fail?
- Task execution quality: does the agent do what you ask?
- Works output quality: is the deliverable useful?
- UI confusion points: where does the interface lose you?
